# M2 — VirtualIP (VIP) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add dpservice-style VirtualIP (1:1 floating IP) to the XDP datapath — outbound SNAT (interface IP → VIP) and inbound DNAT (VIP → interface IP) with correct manual checksum fixups — plus the `CreateVip`/`GetVip`/`DeleteVip` gRPC RPCs, validated on the netns lab and with a tap-fidelity smoke.

**Architecture:** A `VIPS` BPF map holds both directions of the 1:1 mapping. `egress` SNATs the inner IPv4 source before encap; `ingress` DNATs the inner IPv4 destination after decap before delivery. Because XDP lacks `bpf_*_csum_replace`, a `csum` module does RFC-1624 incremental updates of the IPv4 header checksum (and TCP/UDP L4 checksum). gRPC programs the map via the existing `Control`.

**Tech Stack:** Rust + aya/aya-ebpf (rustup `nightly-2026-01-15`, bpf-linker), tonic gRPC, the `env/netns-e2e.sh` lab, QEMU/CirrOS for the tap-fidelity smoke.

**Spec:** `docs/superpowers/specs/2026-06-17-datapath-feature-parity-design.md` (milestone M2).

**Starting point (M1 complete):** map-driven datapath — `egress::try_guest_tx` (PORT_META → ARP → ROUTES → `encap::encap_and_redirect`), `ingress::try_uplink_rx` (decap → INTERFACES → deliver). Maps: `PORT_META`, `INTERFACES`, `ROUTES`, `LOCAL`. gRPC `Control` (`xdp-dp/src/control.rs`) owns the Ebpf + map handles; `CreateInterface`/`CreateRoute` program maps. Single-tenant `vni=0`. The reusable `xdp_inspect` tool + a CirrOS-VM-on-tap recipe exist (from the spike).

## Design decisions locked for M2

- **VIP model:** 1:1. For a guest interface IP `G` with VIP `V`: outbound packets from `G` get src rewritten `G→V` (SNAT); inbound packets to `V` get dst rewritten `V→G` (DNAT), then delivered to `G`'s tap. `G ≠ V`.
- **One bidirectional map** `VIPS: HashMap<VipKey,[u8;4]>` with BOTH entries inserted per VIP: `(vni,G)→V` (egress SNAT lookup by src) and `(vni,V)→G` (ingress DNAT lookup by dst). Egress only ever looks up by src, ingress only by dst, so the directions never collide.
- **Checksums (manual, XDP has no helper):** rewriting a 32-bit IPv4 address requires fixing the IPv4 header checksum and, if present, the TCP/UDP L4 checksum (pseudo-header contains the address). ICMP is unaffected by the address change. UDP checksum `0` means "none" → skip. Use RFC-1624 incremental update.
- **Reachability in the lab:** the VIP is an overlay address routed to the owning hypervisor. The "external client" is modeled as the *other* guest sending to the VIP (an overlay IP with a `ROUTES` entry to the owning hyp) — no separate external path is needed for the PoC.
- **Tap-fidelity:** comprehensive VM/tap testing lands in the ioiab sub-project (needs DHCP). Here, one smoke test (CirrOS VM on a tap, manual IP config via serial) confirms the datapath runs on a real tap; VIP's functional gates stay on the fast netns lab.

```rust
//! Manual incremental checksum updates (XDP has no bpf_l3/l4_csum_replace helpers).

#[inline(always)]
fn fold(mut sum: u32) -> u16 {
    while (sum >> 16) != 0 {
        sum = (sum & 0xffff) + (sum >> 16);
    }
    sum as u16
}

/// RFC 1624 incremental update of a 16-bit ones-complement checksum `check` (in host order,
/// i.e. already `from_be`) when a 32-bit field changes from `old` to `new` (big-endian bytes).
/// Returns the new checksum value (host order) to store back as big-endian.
#[inline(always)]
pub fn csum_replace4(check: u16, old: &[u8; 4], new: &[u8; 4]) -> u16 {
    // HC' = ~( ~HC + (~old as 16-bit words) + (new as 16-bit words) )
    let mut sum: u32 = (!check) as u32;
    sum += (!u16::from_be_bytes([old[0], old[1]])) as u32;
    sum += (!u16::from_be_bytes([old[2], old[3]])) as u32;
    sum += u16::from_be_bytes([new[0], new[1]]) as u32;
    sum += u16::from_be_bytes([new[2], new[3]]) as u32;
    !fold(sum)
}
```

## Task 4: VIP SNAT/DNAT datapath (`vip.rs`) + wiring

**Files:**
- Create: `xdp-dp-ebpf/src/vip.rs`
- Modify: `xdp-dp-ebpf/src/egress.rs`, `xdp-dp-ebpf/src/ingress.rs`, `xdp-dp-ebpf/src/main.rs`

- [ ] **Step 1: Implement `vip.rs`**

```rust
use aya_ebpf::programs::XdpContext;
use xdp_dp_common::VipKey;

use crate::csum::csum_replace4;
use crate::maps::VIPS;
use crate::parse::ETH_LEN;

const IPPROTO_TCP: u8 = 6;
const IPPROTO_UDP: u8 = 17;

/// Rewrite the inner IPv4 SOURCE (egress SNAT) if a VIP exists for it. `ip_off` = offset of the
/// IPv4 header. Returns Ok(()) always (no-op if no VIP). Bounds must already be checked by caller.
#[inline(always)]
pub fn snat_egress(ctx: &XdpContext, ip_off: usize, vni: u32) -> Result<(), ()> {
    rewrite(ctx, ip_off, vni, /*src=*/ true)
}

/// Rewrite the inner IPv4 DEST (ingress DNAT) if a VIP maps to an interface IP.
#[inline(always)]
pub fn dnat_ingress(ctx: &XdpContext, ip_off: usize, vni: u32) -> Result<(), ()> {
    rewrite(ctx, ip_off, vni, /*src=*/ false)
}

#[inline(always)]
fn rewrite(ctx: &XdpContext, ip_off: usize, vni: u32, is_src: bool) -> Result<(), ()> {
    let data = ctx.data();
    let data_end = ctx.data_end();
    if data + ip_off + 20 > data_end {
        return Ok(());
    }
    let p = data as *mut u8;
    let addr_off = ip_off + if is_src { 12 } else { 16 };
    let old = unsafe { core::ptr::read_unaligned(p.add(addr_off) as *const [u8; 4]) };
    let new = match unsafe { VIPS.get(&VipKey { vni, ipv4: old }) } {
        Some(v) => *v,
        None => return Ok(()),
    };
    let ihl = (unsafe { *p.add(ip_off) } & 0x0f) as usize * 4;
    let proto = unsafe { *p.add(ip_off + 9) };
    unsafe {
        // write the new address
        core::ptr::write_unaligned(p.add(addr_off) as *mut [u8; 4], new);
        // fix IPv4 header checksum (offset ip_off+10, big-endian)
        let ipchk = u16::from_be(core::ptr::read_unaligned(p.add(ip_off + 10) as *const u16));
        let ipchk2 = csum_replace4(ipchk, &old, &new);
        core::ptr::write_unaligned(p.add(ip_off + 10) as *mut u16, ipchk2.to_be());
        // fix L4 checksum for TCP/UDP (pseudo-header contains the address)
        let l4 = ip_off + ihl;
        if proto == IPPROTO_TCP && data + l4 + 18 <= data_end {
            let c = u16::from_be(core::ptr::read_unaligned(p.add(l4 + 16) as *const u16));
            let c2 = csum_replace4(c, &old, &new);
            core::ptr::write_unaligned(p.add(l4 + 16) as *mut u16, c2.to_be());
        } else if proto == IPPROTO_UDP && data + l4 + 8 <= data_end {
            let c = u16::from_be(core::ptr::read_unaligned(p.add(l4 + 6) as *const u16));
            if c != 0 {
                let c2 = csum_replace4(c, &old, &new);
                core::ptr::write_unaligned(p.add(l4 + 6) as *mut u16, c2.to_be());
            }
        }
    }
    Ok(())
}
```
> NOTE: this reads `ctx.data()` fresh; the caller passes the IPv4 header offset (`ETH_LEN` once
> the inner frame is at L2, or the appropriate offset at the call site). Keep all accesses
> bounds-checked; the verifier-load gate (Step 4) must pass.

- [ ] **Step 2: Call SNAT in `egress.rs`**

In `egress::try_guest_tx`, AFTER confirming the frame is IPv4 and BEFORE the route lookup/encap,
call `crate::vip::snat_egress(ctx, ETH_LEN, meta.vni)?;` (the inner IPv4 is at `ETH_LEN`). The
route lookup still uses the (unchanged) dst; the encap carries the SNAT'd inner packet.

- [ ] **Step 3: Call DNAT in `ingress.rs`**

In `ingress::try_uplink_rx`, the inner IPv4 dst is read at `ETH_LEN + IPV6_LEN + 16` to find the
interface; for VIP, first resolve the real interface: look up `VIPS[(0, inner_dst)]` — if it maps
to `G`, use `G` for the `INTERFACES` lookup. THEN after `adjust_head(+40)` and writing the inner
Ethernet, call `crate::vip::dnat_ingress(ctx, ETH_LEN, 0)` to rewrite the inner dst `V→G` + fix
checksums (the IPv4 is now at `ETH_LEN`). Order: (1) read inner_dst=V; (2) `G = VIPS[(0,V)]`
else V; (3) `iface = INTERFACES[(0,G)]`; (4) adjust_head; (5) write inner eth; (6)
`dnat_ingress`; (7) redirect to `iface.tap_ifindex`.
