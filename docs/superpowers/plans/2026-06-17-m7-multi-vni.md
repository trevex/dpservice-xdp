# M7 — Multi-VNI Tenancy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the datapath genuinely multi-tenant — multiple VNIs whose guests may use **overlapping overlay IPv4 addresses** — by adopting dpservice's per-interface underlay-IPv6 addressing: each interface gets a unique underlay IPv6 from the hypervisor's /64, `uplink_rx` resolves the **outer IPv6 destination → (vni, tap)** to deliver, every feature lookup is keyed by the real VNI, and conntrack keys include the VNI so overlapping flows never collide.

**Architecture:** A new `UNDERLAY` map (`underlay_ipv6 → {vni, tap_ifindex, guest_mac}`) is the ingress delivery table: the outer IPv6 dst of a decapped packet uniquely identifies the destination interface and its VNI. The outer eth dst MAC still performs L2 delivery to the peer uplink (flat-L2), so per-interface underlay IPs are pure datapath identifiers (XDP intercepts before the kernel IPv6 stack — no kernel addresses needed). `PortMeta` carries the source interface's underlay IPv6 (outer src on encap). `CtKey` gains a `vni` field. `CreateInterface` assigns the per-interface underlay IPv6 (returned as `underlay_route`); `CreateRoute` nexthops point at the destination interface's underlay IPv6.

**Tech Stack:** Rust + aya/aya-ebpf (rustup `nightly-2026-01-15`, bpf-linker), tonic gRPC, `env/netns-e2e.sh`.

**Spec:** `docs/superpowers/specs/2026-06-17-full-parity-gap-design.md` (§4.2; milestone M7).

**Starting point (M1–M6 complete):** single-tenant `vni=0`. Maps already key on `vni` (`IfaceKey`, `RouteKey`, `VipKey`, `LbKey`, `NatKey`, `PortMeta.vni`) but only `0` is written. `egress::try_guest_tx`: `meta=PORT_META[ingress_ifindex]` (carries `vni`) → ARP → conntrack apply/touch + egress firewall → VIP snat(vni) → ROUTES[(vni,dst)] → NAT egress(vni) → DEFAULT-ensure → `encap_and_redirect(ctx, local, route, inner_len)` (outer src = `local.underlay_ipv6` hypervisor-wide). `ingress::try_uplink_rx`: reads inner dst → VIP target resolve `(vni=0)` → `lb_select_dnat(ctx, off, 0)` → NAT reverse (generic conntrack) → `INTERFACES[(vni=0, deliver_ip)]` → firewall ingress → write inner eth → VIP dnat `(0)` → redirect. `CtKey{src_ip,dst_ip,src_port,dst_port,proto,_pad[3]}`(16). `conntrack::ct_key(data,data_end,ip_off)->Option<CtKey>` (no vni). `encap.rs` uses `local.underlay_ipv6` as outer src, `route.nexthop_ipv6` as outer dst. 9 e2e tests.

## Design decisions locked for M7

- **Per-interface underlay IPv6 (dpservice model):** the hypervisor owns a /64; each interface is assigned a unique /128 from it. That /128 is the interface's identity on the underlay and is returned by `CreateInterface` as `underlay_route`. Overlapping overlay IPv4 across VNIs is disambiguated by the distinct underlay /128.
- **Ingress delivers by OUTER IPv6 dst:** `uplink_rx` reads the outer IPv6 dst → `UNDERLAY[outer_dst] = {vni, tap_ifindex, guest_mac}`. This yields the VNI used for all subsequent inner-IP feature lookups, and the delivery tap (overridden only by LB to the backend's tap). `INTERFACES[(vni,ipv4)]` is retained for LB-backend resolution.
- **Outer eth MAC does L2 delivery (unchanged):** outer eth dst = `local.gateway_mac` (peer uplink/ToR MAC). The underlay /128s need no kernel configuration in the flat-L2 lab; they are datapath identifiers. (In production the underlay is routed and the /128s are real, but the datapath logic is identical.)
- **Outer src = source interface underlay:** `PortMeta` gains `underlay_ipv6`; encap sets outer src to it (per-interface fidelity).
- **VNI in conntrack:** `CtKey` gains `vni: u32` so overlapping-IP flows in different VNIs never collide. `ct_key` takes a `vni` argument; egress uses `meta.vni`, ingress uses the `UNDERLAY`-resolved vni.
- **Routes carry the destination interface underlay:** `CreateRoute`/`--remote` nexthop = the remote interface's underlay /128 (what its `CreateInterface` returned), not a hypervisor-wide address.
- **Backward compatible:** all existing single-tenant tests keep passing (they become "tenant vni=0 with per-interface underlay IPs"); a second tenant (vni=100) with overlapping IPs is added to prove isolation.

## Task 4: Ingress delivers by outer IPv6 dst (UNDERLAY) + VNI threading

**Files:** Modify `xdp-dp-ebpf/src/ingress.rs`

- [ ] **Step 1: Resolve (vni, tap, guest_mac) from the outer IPv6 dst**
In `try_uplink_rx`, after the IPv6/IPIP checks and BEFORE the inner-dst/VIP/LB logic, read the outer IPv6 dst (16 bytes at `ETH_LEN + 24`) and resolve the destination interface:
```rust
    let outer_dst = unsafe { core::ptr::read_unaligned(p.add(ETH_LEN + 24) as *const [u8; 16]) };
    let u = match unsafe { crate::maps::UNDERLAY.get(&outer_dst) } {
        Some(u) => *u,
        None => return Ok(xdp_action::XDP_PASS),
    };
    let vni = u.vni;
```

- [ ] **Step 2: Thread `vni` and deliver by the underlay-resolved interface**
Replace the existing `target`/`INTERFACES[(vni=0, deliver_ip)]` resolution with: the VIP/LB/NAT feature lookups use `vni`; the delivery tap/guest_mac come from `u` (the underlay-resolved interface), EXCEPT when LB selected a backend — then resolve the backend's interface via `INTERFACES[(vni, backend)]`:
```rust
    // Inner-IP feature lookups now use the real vni.
    let lb_backend = crate::lb::lb_select_dnat(ctx, ETH_LEN + IPV6_LEN, vni);
    let nat_guest = if lb_backend.is_none() {
        let d = ctx.data(); let de = ctx.data_end();
        match crate::conntrack::ct_key(d, de, ETH_LEN + IPV6_LEN, vni) {
            Some(key) => match unsafe { crate::maps::CONNTRACK.get(&key) } {
                Some(e) if e.flags & CT_REWRITE_DST != 0 => {
                    let mut e = *e;
                    crate::conntrack::ct_apply(ctx, ETH_LEN + IPV6_LEN, &e);
                    crate::conntrack::ct_touch(ctx, ETH_LEN + IPV6_LEN, &key, &mut e);
                    Some(e.xlate_ip)
                }
                _ => None,
            },
            None => None,
        }
    } else { None };
    // Delivery interface: LB backend's iface if LB matched, else the underlay-resolved iface.
    let (tap_ifindex, guest_mac) = match lb_backend {
        Some(backend) => {
            let bi = unsafe { crate::maps::INTERFACES.get(&IfaceKey { vni, ipv4: backend }) }.ok_or(())?;
            if bi.is_local == 0 { return Ok(xdp_action::XDP_PASS); }
            (bi.tap_ifindex, bi.guest_mac)
        }
        None => (u.tap_ifindex, u.guest_mac),
    };
```
Remove the old `inner_dst`/`target`/VIPS-target block that resolved the delivery interface by inner IP. (VIP **DNAT** of the inner header still happens post-adjust via `vip::dnat_ingress(ctx, ETH_LEN, vni)` — keep that, now with `vni`.) The firewall ingress + DEFAULT tracking blocks now use `vni` in their `ct_key(...)` calls and `tap_ifindex` for the firewall.
