# M6 — Stateful Firewall Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add dpservice-style **stateful, per-interface firewall** — priority-ordered accept/drop rules matched on direction + src/dst prefix + protocol + port ranges / ICMP type-code, evaluated once per flow and enforced on both the egress (source interface) and ingress (destination interface) XDP hooks, with statefulness provided by the unified conntrack from M5 — plus `CreateFirewallRule`/`GetFirewallRule`/`DeleteFirewallRule`/`ListFirewallRules` gRPC.

**Architecture:** Rules live in BPF maps keyed by interface. The datapath evaluates the firewall only for a flow's **first packet** (no conntrack entry yet); established flows ride the M5 conntrack and are allowed without re-evaluation, which gives stateful return-traffic admission. `guest_tx` enforces the source interface's **EGRESS** rules; `uplink_rx` enforces the destination interface's **INGRESS** rules; together they reproduce dpservice's "both source-egress and dest-ingress must accept". Enforcement (actual `XDP_DROP`) is **ON by default**, switchable via `FW_CONFIG`.

**Tech Stack:** Rust + aya/aya-ebpf (rustup `nightly-2026-01-15`, bpf-linker), tonic gRPC, `env/netns-e2e.sh`.

**Spec:** `docs/superpowers/specs/2026-06-17-full-parity-gap-design.md` (§4.1; milestone M6). Grounded in dpservice `src/dp_firewall.c` (`dp_get_firewall_action`, `dp_get_egress_action`, `dp_is_rule_matching`), `src/nodes/firewall_node.c`, `include/dp_firewall.h`.

**Starting point (M1–M5 complete):** unified `CONNTRACK<CtKey,CtEntry>` tracks **every** flow (DEFAULT + LB/NAT), with TCP state, `last_seen`, and a userspace GC. `CtEntry` already reserves a `fwall_action: u8` field (written `0` today). `egress::try_guest_tx`: `meta = PORT_META[ingress_ifindex]` → ARP → conntrack apply/touch (top) → VIP snat → ROUTES → NAT egress → DEFAULT-ensure → encap. `ingress::try_uplink_rx`: decap-prep → VIP target resolve → `lb::lb_select_dnat` → NAT reverse (generic) → INTERFACES[deliver_ip] (yields `iface.tap_ifindex`) → write inner eth → DEFAULT touch/ensure → redirect. `conntrack::{ct_key,ct_apply,ct_touch,ct_ensure_default,now}`. 9 passing e2e tests.

## Design decisions locked for M6

- **Whitelist semantics (dpservice `dp_get_egress_action`):** in a given direction, an interface with **zero rules ⇒ ACCEPT**; with ≥1 rule ⇒ the first matching rule's action (priority order), and **no match ⇒ DROP**.
- **Deliberate deviation:** dpservice's *ingress* is default-DROP even when empty (metalnet always programs rules). We make **empty-ingress ⇒ ACCEPT** too, so a rule-less interface is not bricked and existing lab flows keep working. Documented; to get exact dpservice default-deny, program an explicit low-priority deny rule.
- **Statefulness via conntrack entry existence (M5):** the firewall evaluates a flow **only on its first packet** (no `CONNTRACK` entry for that 5-tuple). Established flows — including **return traffic**, because `ct_ensure_default` is extended to seed BOTH directions — already have an entry and are allowed without re-evaluation. This reproduces dpservice's per-flow `DP_FLOW_FLAG_FIREWALL` caching using entry existence rather than a cached byte. (The `fwall_action` byte is still written for inspection/parity, but correctness rests on entry existence.)
- **Two-hook split:** `guest_tx` evaluates the **source** interface's EGRESS rules (source = `meta` ifindex); `uplink_rx` evaluates the **destination** interface's INGRESS rules (dest = resolved delivery iface). Combined = dpservice's egress∧ingress requirement. Each hook evaluates on the post-NAT/DNAT 5-tuple (firewall runs after NAT in dpservice's graph too).
- **Enforcement default ON:** a single-entry `FW_CONFIG` map holds `enforce: u32` (default `1`), set from `--firewall-enforce <bool>` / `XDP_DP_FIREWALL_ENFORCE`. When `0`, the datapath evaluates + records the decision but never drops (upstream-matching permissive mode).
- **Bounded rule scan:** up to `FW_MAX_RULES = 16` rules per interface per direction are scanned in-datapath (a bounded loop the verifier accepts). Userspace assigns each rule a `rule_id` and maps it to a `(ifindex, slot)`.
- **Match fields:** `proto` (0 = any), src/dst IPv4 masked by an explicit 4-byte mask, TCP/UDP src+dst port **ranges** (`min..=max`; `0..=65535` = any), ICMP `type`/`code` (`0xffff` = any). IPv6-tenant firewalling is out of scope until M15.

/// Max firewall rules scanned per interface per direction in the datapath (bounded loop).
pub const FW_MAX_RULES: u32 = 16;

/// Firewall rule slot key: (interface ifindex, slot index 0..FW_MAX_RULES).
#[repr(C)]
#[derive(Copy, Clone, Eq, PartialEq, Hash, Debug, Default)]
pub struct FwRuleKey {
    pub ifindex: u32,
    pub idx: u32,

/// A single firewall rule (fixed-size POD). Ports are inclusive ranges (0..=65535 = any);
/// icmp_type/icmp_code 0xffff = any; proto 0 = any; action 1=accept/0=drop; direction
/// 1=egress/0=ingress; enabled 1 = slot in use.
#[repr(C)]
#[derive(Copy, Clone, Eq, PartialEq, Debug, Default)]
pub struct FwRule {
    pub src_ip: [u8; 4],
    pub src_mask: [u8; 4],
    pub dst_ip: [u8; 4],
    pub dst_mask: [u8; 4],
    pub src_port_min: u16,
    pub src_port_max: u16,
    pub dst_port_min: u16,
    pub dst_port_max: u16,
    pub icmp_type: u16,
    pub icmp_code: u16,
    pub proto: u8,
    pub action: u8,
    pub direction: u8,
    pub enabled: u8,
}

/// Per-interface rule counts (so empty-direction => ACCEPT can be decided cheaply).
#[repr(C)]
#[derive(Copy, Clone, Eq, PartialEq, Debug, Default)]
pub struct FwMeta {
    pub ingress_count: u32,
    pub egress_count: u32,
}

pub const FW_DIR_INGRESS: u8 = 0;
pub const FW_DIR_EGRESS: u8 = 1;
pub const FW_ACTION_DROP: u8 = 0;
pub const FW_ACTION_ACCEPT: u8 = 1;

- [ ] **Step 3: Implement `fw_rule_matches` (lives in the crate root so both ebpf + host use it)**
```rust
/// Pure firewall match (no_std; used by the datapath and host-tested). Returns true if `r` matches
/// the packet selectors. `icmp_type`/`icmp_code` are ignored unless `proto == 1`.
#[inline]
pub fn fw_rule_matches(
    r: &FwRule,
    src: &[u8; 4],
    dst: &[u8; 4],
    proto: u8,
    sport: u16,
    dport: u16,
    icmp_type: u16,
    icmp_code: u16,
) -> bool {
    if r.enabled == 0 {
        return false;
    }
    if r.proto != 0 && r.proto != proto {
        return false;
    }
    let mut i = 0;
    while i < 4 {
        if src[i] & r.src_mask[i] != r.src_ip[i] & r.src_mask[i] {
            return false;
        }
        if dst[i] & r.dst_mask[i] != r.dst_ip[i] & r.dst_mask[i] {
            return false;
        }
        i += 1;
    }
    match proto {
        6 | 17 => {
            sport >= r.src_port_min
                && sport <= r.src_port_max
                && dport >= r.dst_port_min
                && dport <= r.dst_port_max
        }
        1 => {
            (r.icmp_type == 0xffff || icmp_type == r.icmp_type)
                && (r.icmp_code == 0xffff || icmp_code == r.icmp_code)
        }
        _ => true,
    }
}
```

```rust
use xdp_dp_common::{
    fw_rule_matches, FwMeta, FwRule, FwRuleKey, FW_ACTION_ACCEPT, FW_ACTION_DROP, FW_MAX_RULES,
};

use crate::maps::{FW_CONFIG, FW_META, FW_RULES};
use crate::parse::l4_ports;

/// Read whether enforcement is enabled (FW_CONFIG[0] != 0; default 1 when unset).
#[inline(always)]
pub fn fw_enforcing() -> bool {
    match FW_CONFIG.get(0) {
        Some(v) => *v != 0,
        None => true,
    }
}

/// Extract ICMP type/code for an IPv4 packet at `ip_off` (0,0 if not ICMP / OOB). Requires IHL==5.
#[inline(always)]
fn icmp_type_code(data: usize, data_end: usize, ip_off: usize) -> (u16, u16) {
    let p = data as *const u8;
    if data + ip_off + 20 > data_end {
        return (0, 0);
    }
    if unsafe { *p.add(ip_off) } & 0x0f != 5 || unsafe { *p.add(ip_off + 9) } != 1 {
        return (0, 0);
    }
    let l4 = ip_off + 20;
    if data + l4 + 2 > data_end {
        return (0, 0);
    }
    (
        unsafe { *p.add(l4) } as u16,
        unsafe { *p.add(l4 + 1) } as u16,
    )
}

/// Evaluate the firewall for a packet at `ip_off` against interface `ifindex` rules in `dir`
/// (FW_DIR_*). Whitelist semantics: zero rules in this direction => ACCEPT; else first matching
/// rule's action; no match => DROP. Returns FW_ACTION_ACCEPT / FW_ACTION_DROP.
#[inline(always)]
pub fn fw_eval_dir(data: usize, data_end: usize, ip_off: usize, ifindex: u32, dir: u8) -> u8 {
    // Empty direction => accept.
    let meta: FwMeta = match unsafe { FW_META.get(&ifindex) } {
        Some(m) => *m,
        None => return FW_ACTION_ACCEPT,
    };
    let count = if dir == xdp_dp_common::FW_DIR_EGRESS {
        meta.egress_count
    } else {
        meta.ingress_count
    };
    if count == 0 {
        return FW_ACTION_ACCEPT;
    }
    if data + ip_off + 20 > data_end {
        return FW_ACTION_ACCEPT;
    }
    let p = data as *const u8;
    let src = unsafe { core::ptr::read_unaligned(p.add(ip_off + 12) as *const [u8; 4]) };
    let dst = unsafe { core::ptr::read_unaligned(p.add(ip_off + 16) as *const [u8; 4]) };
    let (proto, sport, dport) = match l4_ports(data, data_end, ip_off) {
        Some(v) => v,
        None => (unsafe { *p.add(ip_off + 9) }, 0u16, 0u16),
    };
    let (itype, icode) = icmp_type_code(data, data_end, ip_off);
    // Bounded scan; first matching rule for this direction wins.
    let mut idx: u32 = 0;
    while idx < FW_MAX_RULES {
        if let Some(r) = unsafe { FW_RULES.get(&FwRuleKey { ifindex, idx }) } {
            let r: FwRule = *r;
            if r.direction == dir
                && fw_rule_matches(&r, &src, &dst, proto, sport, dport, itype, icode)
            {
                return r.action;
            }
        }
        idx += 1;
    }
    FW_ACTION_DROP
}
```
