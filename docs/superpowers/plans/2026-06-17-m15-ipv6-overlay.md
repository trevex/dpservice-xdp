# M15 — Dual-Stack IPv6 Overlay + Neighbor Discovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Carry **IPv6 overlay traffic** end-to-end — a dual-stack guest sends/receives native IPv6 packets over the IP-in-IPv6 overlay (IPv6-in-IPv6, outer next-header 41), routed by an IPv6 LPM table, with the gateway resolved by an **in-datapath Neighbor Discovery responder** (the IPv6 analog of the existing ARP responder). This is what ioiab's `--enable-ipv6-overlay` requires.

**Scope:** dual-stack **forwarding** (v6 guest ↔ v6 guest over the overlay) + **ND**. Out of scope here (forwarding only for v6): VIP / LB / NAT-GW / firewall / conntrack on IPv6 inner packets — those stay IPv4-only (the v6 path is decap/deliver + encap/route). **NAT64 is deferred** (per the v6→v4 discussion: only needed for IPv6-only guests reaching IPv4-only destinations; dual-stack guests use their IPv4 address + M4 NAT-GW for v4 destinations).

**Architecture:** Egress (`guest_tx`) dispatches on the inner Ethertype: `0x0800` → today's IPv4 pipeline; `0x86DD` → the v6 path (answer ND for the gateway, else look up the inner IPv6 dst in `ROUTES6` and encap with inner-proto 41). Ingress (`uplink_rx`) dispatches on the outer next-header: `4` (IPIP) → today's IPv4 pipeline; `41` (IPV6) → decap + deliver the inner IPv6 frame to the `UNDERLAY`-resolved interface (delivery is by outer IPv6 dst, family-agnostic). `encap_and_redirect` gains an `inner_proto` parameter.

**Tech Stack:** Rust + aya/aya-ebpf (rustup `nightly-2026-01-15`, bpf-linker), tonic gRPC, `env/netns-e2e.sh`.

**Spec:** `docs/superpowers/specs/2026-06-17-full-parity-gap-design.md` (§4.10; milestone M15, NAT64 deferred). dpservice `ipv6_nd_node` (ND responder) + `ipv6_lookup_node` (v6 LPM) + `ipip_encap`/`ipip_decap` (next-header 41 for v6 inner).

**Starting point (M1–M10, M12, M13):** `parse.rs` has `ETH_LEN=14, IPV6_LEN=40, ETH_P_IP=0x0800, ETH_P_IPV6=0x86DD, ETH_P_ARP=0x0806, IPPROTO_IPIP=4`. `arp_nd::try_arp_reply(ctx, meta)` answers ARP for `meta.gateway_ipv4` → `XDP_TX`. `egress::try_guest_tx`: ARP → `if ethertype != ETH_P_IP { return XDP_PASS }` → v4 pipeline → `encap_and_redirect(ctx, local, &meta.underlay_ipv6, route, inner_len)` (builds outer IPv6, inner-proto = `IPPROTO_IPIP`). `ingress::try_uplink_rx`: `if *(p+ETH_LEN+6) != IPPROTO_IPIP { return XDP_PASS }` → resolve `outer_dst → UNDERLAY → (vni, tap, guest_mac)` → v4 features → decap (`adjust_head(+IPV6_LEN)`) → write inner eth (`guest_mac`, `GW_MAC`, `ETH_P_IP`) → `bpf_redirect(tap)`. `ROUTES: LpmTrie<RouteLpmData, RouteValue>` (`RouteLpmData{vni:[u8;4], ipv4:[u8;4]}`). `PortMeta{vni, guest_ipv4, gateway_ipv4, guest_mac, _pad, underlay_ipv6}`. 13 e2e tests; the IPv6 path is additive (no v6 config ⇒ unchanged).

## Design decisions locked for M15

- **IPv6-in-IPv6, outer next-header 41** (`IPPROTO_IPV6`). The existing `encap_and_redirect` already writes the outer IPv6 header; parameterize the inner-proto byte (4 vs 41) and the inner length.
- **`ROUTES6`** = a second LPM trie keyed by `RouteLpmData6{vni:[u8;4], ipv6:[u8;16]}` (20-byte data; prefix_len = 32 + v6_prefix; lookup prefix_len = 160). Separate from the v4 `ROUTES` to keep key sizes clean.
- **Delivery is family-agnostic.** `UNDERLAY[outer_dst] → (vni, tap, guest_mac)` is reused unchanged; the inner family only changes the inner-eth Ethertype on decap.
- **ND responder** answers ICMPv6 Neighbor Solicitation (type 135) for `meta.gateway_ipv6` with a Neighbor Advertisement (type 136, solicited+override, target-LL-addr = `GW_MAC`), recomputing the ICMPv6 checksum over the v6 pseudo-header. `PortMeta` gains `gateway_ipv6`.
- **v6 inner skips all v4 features.** No VIP/LB/NAT/firewall/conntrack on v6 (those read IPv4 offsets). The v6 path is pure forward/deliver. (v6 stateful features are a documented follow-on.)
- **Additive / opt-in.** No v6 config ⇒ `guest_tx` v6 branch is never hit (no v6 guests) and `uplink_rx` proto-41 branch never fires. The 13 e2e tests are unchanged.

## Task 3: ND responder (`arp_nd.rs::try_nd_reply`)

**Files:** Modify `xdp-dp-ebpf/src/arp_nd.rs`

- [ ] **Step 1: Implement ND (ICMPv6 NS → NA)**
A Neighbor Solicitation for `meta.gateway_ipv6` is: Eth(14) + IPv6(40, next-header 58) + ICMPv6 NS (type=135 @ +0, code @ +1, cksum @ +2, reserved @ +4..8, target @ +8..24) + optional source-LL-addr option. Rewrite it in place into a Neighbor Advertisement (type 136) from the gateway:
```rust
use crate::parse::{write16, IPPROTO_ICMPV6, IPV6_LEN};

const ND_NS: u8 = 135;
const ND_NA: u8 = 136;

/// One's-complement checksum over `len` bytes at `ptr`, plus an initial `sum` (for the pseudo-header).
#[inline(always)]
unsafe fn csum16(mut sum: u32, ptr: *const u8, len: usize) -> u16 {
    let mut i = 0;
    while i + 1 < len {
        sum += u16::from_be(core::ptr::read_unaligned(ptr.add(i) as *const u16)) as u32;
        i += 2;
    }
    if i < len {
        sum += (*ptr.add(i) as u32) << 8;
    }
    while sum >> 16 != 0 {
        sum = (sum & 0xffff) + (sum >> 16);
    }
    !(sum as u16)
}

/// If the frame is an ICMPv6 Neighbor Solicitation for `meta.gateway_ipv6`, rewrite it in place
/// into a solicited Neighbor Advertisement from GW_MAC and return Some(XDP_TX). The NS/NA are a
/// FIXED size here (NS with a source-LL-addr option = 40+32 = 72 inner bytes) so all accesses are
/// constant-offset (verifier-friendly).
#[inline(always)]
pub fn try_nd_reply(ctx: &XdpContext, meta: &PortMeta) -> Option<u32> {
    let data = ctx.data();
    let data_end = ctx.data_end();
    // Eth(14)+IPv6(40)+ICMPv6 NS(24)+option(8) = 86
    if data + ETH_LEN + IPV6_LEN + 32 > data_end {
        return None;
    }
    let p = data as *mut u8;
    let ethertype = u16::from_be(unsafe { core::ptr::read_unaligned(p.add(12) as *const u16) });
    if ethertype != crate::parse::ETH_P_IPV6 {
        return None;
    }
    let ip = unsafe { p.add(ETH_LEN) };
    if unsafe { *ip.add(6) } != IPPROTO_ICMPV6 {
        return None;
    }
    let icmp = unsafe { p.add(ETH_LEN + IPV6_LEN) };
    if unsafe { *icmp } != ND_NS {
        return None;
    }
    let target = unsafe { core::ptr::read_unaligned(icmp.add(8) as *const [u8; 16]) };
    if target != meta.gateway_ipv6 {
        return None;
    }
    let req_mac = unsafe { core::ptr::read_unaligned(p as *const [u8; 6]) };
    let req_src = unsafe { core::ptr::read_unaligned(ip.add(8) as *const [u8; 16]) };
    unsafe {
        // Ethernet: dst = requester, src = GW_MAC.
        write6(p, &req_mac);
        write6(p.add(6), &GW_MAC);
        // IPv6: src = gateway (the advertised target), dst = requester's src. hop limit 255.
        write16(ip.add(8), &meta.gateway_ipv6);
        write16(ip.add(24), &req_src);
        *ip.add(7) = 255;
        // payload_len = 32 (NA 24 + target-LL option 8)
        core::ptr::write_unaligned(ip.add(4) as *mut u16, 32u16.to_be());
        // ICMPv6 NA: type=136, code=0, flags = solicited|override (0x60), reserved 0, target unchanged.
        *icmp = ND_NA;
        *icmp.add(1) = 0;
        core::ptr::write_unaligned(icmp.add(2) as *mut u16, 0); // cksum cleared for recompute
        *icmp.add(4) = 0x60;
        *icmp.add(5) = 0;
        *icmp.add(6) = 0;
        *icmp.add(7) = 0;
        // target address @ +8 stays = gateway. Option @ +24: type=2 (target LL addr), len=1, GW_MAC.
        *icmp.add(24) = 2;
        *icmp.add(25) = 1;
        write6(icmp.add(26), &GW_MAC);
        // ICMPv6 checksum over pseudo-header (src,dst,len=32,next=58) + message(32 bytes).
        let mut sum: u32 = 0;
        let mut k = 0;
        while k < 16 {
            sum += u16::from_be(core::ptr::read_unaligned(ip.add(8 + k) as *const u16)) as u32; // src
            sum += u16::from_be(core::ptr::read_unaligned(ip.add(24 + k) as *const u16)) as u32; // dst
            k += 2;
        }
        sum += 32u32; // upper-layer length
        sum += IPPROTO_ICMPV6 as u32; // next header
        let cks = csum16(sum, icmp as *const u8, 32);
        core::ptr::write_unaligned(icmp.add(2) as *mut u16, cks.to_be());
    }
    Some(xdp_action::XDP_TX)
}
```
(Imports: `try_nd_reply` reuses `GW_MAC`, `write6`, `ETH_LEN`, `xdp_action` already in `arp_nd.rs`; add `write16`, `IPPROTO_ICMPV6`, `IPV6_LEN` and `crate::parse::ETH_P_IPV6`.)

```rust
use aya_ebpf::{bindings::xdp_action, helpers::bpf_xdp_adjust_head, programs::XdpContext};
use xdp_dp_common::{PortMeta, RouteLpmData6, RouteValue};

use crate::arp_nd::GW_MAC;
use crate::encap::encap_and_redirect;
use crate::maps::{LOCAL, ROUTES6, UNDERLAY};
use crate::parse::{write6, ETH_LEN, ETH_P_IPV6, IPPROTO_IPV6, IPV6_LEN};

/// Egress for an inner IPv6 frame: route the inner v6 dst via ROUTES6 and encap (inner-proto 41).
/// Returns the XDP action, or Err to PASS.
#[inline(always)]
pub fn v6_guest_tx(ctx: &XdpContext, meta: &PortMeta) -> Result<u32, ()> {
    let data = ctx.data();
    let data_end = ctx.data_end();
    if data + ETH_LEN + IPV6_LEN > data_end {
        return Ok(xdp_action::XDP_PASS);
    }
    let p = data as *const u8;
    // inner IPv6 dst at ETH_LEN + 24
    let dst = unsafe { core::ptr::read_unaligned(p.add(ETH_LEN + 24) as *const [u8; 16]) };
    let route = ROUTES6
        .get(&aya_ebpf::maps::lpm_trie::Key::new(
            160,
            RouteLpmData6 { vni: meta.vni.to_be_bytes(), ipv6: dst },
        ))
        .ok_or(())?;
    let inner_len = (data_end - data - ETH_LEN) as u16;
    let local = LOCAL.get(0).ok_or(())?;
    encap_and_redirect(ctx, local, &meta.underlay_ipv6, route, inner_len, IPPROTO_IPV6)
}

/// Ingress for an inner IPv6 frame (outer next-header 41): resolve the dest interface by outer
/// IPv6 dst, decap, write the inner Ethernet (Ethertype IPv6), and redirect to the tap.
#[inline(always)]
pub fn v6_uplink_rx(ctx: &XdpContext) -> Result<u32, ()> {
    let data = ctx.data();
    let data_end = ctx.data_end();
    if data + ETH_LEN + IPV6_LEN + 40 > data_end {
        return Ok(xdp_action::XDP_PASS);
    }
    let p = data as *const u8;
    let outer_dst = unsafe { core::ptr::read_unaligned(p.add(ETH_LEN + 24) as *const [u8; 16]) };
    let u = match unsafe { UNDERLAY.get(&outer_dst) } {
        Some(u) => *u,
        None => return Ok(xdp_action::XDP_PASS),
    };
    if unsafe { bpf_xdp_adjust_head(ctx.ctx, IPV6_LEN as i32) } != 0 {
        return Err(());
    }
    let data = ctx.data();
    let data_end = ctx.data_end();
    if data + ETH_LEN > data_end {
        return Err(());
    }
    let q = data as *mut u8;
    unsafe {
        write6(q, &u.guest_mac);
        write6(q.add(6), &GW_MAC);
        core::ptr::write_unaligned(q.add(12) as *mut u16, ETH_P_IPV6.to_be());
    }
    Ok(unsafe { aya_ebpf::helpers::bpf_redirect(u.tap_ifindex, 0) } as u32)
}
```
